Data format description.

  • Data format is a stricly JSON format for mobile developers.
  • Each Data message should contain an Application token.

Sign in and Sign up.

There are two levels of authentication. The first level - is Application token auth, The second is - User token auth.

  • Application token - is a token that was given you by API vendor. And identifies your application as a valid one for using this API.
  • Session token - is a token that identifies the wallet's owner and that the /signin method returns.

Methods:

/signin

Request method: POST

Description: This method is for signing in an existing user account. Optionaly the user can sign in with his facebook credentials using Facebook API. Use this method for saving user token for further request authorization.

Header:
app-token:test-token;
Data:

{
"signin":
{
"accountid":"vk@onlinepay.com" [email | phone | fid],
"password":"sakjh43589jf"
}
}

Returns:

{
"session":
{
"result":[0|1|N],
"message":"ok",
"expiration":"2014-04-04 23:44"
}
}
